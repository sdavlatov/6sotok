## 1. Что такое 6sotok.kz

Специализированный маркетплейс земельных участков и готового бизнеса в Казахстане.  
Продавцы (физлица и агентства) размещают объявления бесплатно.  
Покупатели находят через каталог или карту и выходят напрямую на продавца (звонок / WhatsApp).  
Платформа в сделке не участвует — только соединяет стороны.

**Домен:** 6sotok.kz  
**Prod:** https://6sotok.vercel.app  
**Admin:** https://6sotok.vercel.app/admin  
**Юрлицо:** ТОО «6sotok», Алматы, Казахстан  
**Основан:** 2024. Раздел бизнеса добавлен в 2025.

| Слой | Технология |

| Сервер (dev) | systemd user service `6sotok-dev.service` в WSL2 |

## 3. Инфраструктура

### Dev (локально)
```
Next.js:   systemd --user start 6sotok-dev.service   → http://localhost:3000
Postgres:  docker container "postgres"               → localhost:5432
Медиа:     Vercel Blob (общий с prod!)
Schema:    push: true — Payload сам мигрирует локальный Postgres
```

### Prod (Vercel)
```
Next.js:   Vercel serverless                         → https://6sotok.vercel.app
Postgres:  Neon (непулированный URL)
Медиа:     Vercel Blob
Schema:    push: false — изменения вручную через Neon SQL Editor
```

### Правило изменения схемы БД
Любое новое поле в коллекции требует двух шагов:
1. Локально — автоматически через Payload push
2. Neon (prod) — вручную через SQL Editor: `ALTER TABLE listings ADD COLUMN IF NOT EXISTS ...`

Маппинг: Payload camelCase → Postgres snake_case (`dealType` → `deal_type`)

### Команды
```bash
systemctl --user status 6sotok-dev.service    # статус
systemctl --user restart 6sotok-dev.service   # перезапуск (нужен после изменений через IDE)
journalctl --user -u 6sotok-dev.service -f    # логи
git push origin main                          # деплой на prod (Vercel автодеплой)
```

## 7. API и бизнес-логика

### Эндпоинты
| URL | Метод | Описание |
|-----|-------|---------|
| `/api/listings` | GET | Получение по IDs (`?ids=1,2,3`) — для избранного |
| `/api/listings` | POST | Создание объявления из формы |
| `/api/map-listings` | GET | Облегчённые данные для карты |
| `/api/view` | POST | Инкремент счётчика просмотров |
| `/api/media` | POST | Загрузка медиафайлов (Payload route) |
| `/api/listings/bulk-publish` | POST | Массовая публикация (только admin) |

### Ключевые паттерны
- **Slug:** автоген `listing-{timestamp}` → afterChange hook заменяет на `String(doc.id)`
- **Избранное:** localStorage `6sotok_bookmarks` = `string[]` ID + DOM событие `bookmarks-updated`
- **Видео:** определяется по расширению `/\.(mp4|mov|webm|ogv|m4v)$/i`
- **Hydration fix:** компоненты с видео используют `mounted` state + `useEffect`
- **Карта объявления:** при открытии запрос к Overpass API (POI в радиусе 3-120 км), кеш 24ч
- **Сравнение:** IDs в URL `?ids=1,2,3`, до 4 штук
- **Форма:** единый `fd` state, `set(key, value)` / `toggle(key)`, форматирование цены через `fmtPrice`/`rawPrice`

### Важные ограничения
- **НЕ** делать `fetch('http://localhost:3000/api/...')` из серверных компонентов — только `getPayload({ config })`
- **НЕ** объявлять `interface Window { L? }` нигде кроме `map-view.tsx`
- **НЕ** импортировать `from 'leaflet'` — типы локальные в `map-view.tsx`
- **НЕ** передавать `Listing[]` как пропс в клиентские компоненты главной — только скалярные данные

### Цвета (globals.css)
```css
--color-primary:       #066F36   /* основной зелёный, бренд */
--color-primary-light: #2CA64E   /* hover-акценты */

--color-primary-dark:  #022c15   /* тёмный вариант */
--color-accent:        #A3D2F0   /* небесно-голубой */
--color-muted:         #a1a1aa   /* zinc-400 */
--color-border:        #e4e4e7   /* zinc-200 */
--color-card:          #ffffff
--color-background:    #fafafa
```

### Шрифты
- **Основной:** Inter (next/font/google)
- Заголовки: `font-black tracking-tight` (-0.04em до -0.05em)
- Тело: `text-[15px] leading-relaxed text-zinc-700`
- Числа/цены: `font-bold tabular-nums` (не font-black в карточках!)

### Ключевые правила
- `rounded-2xl` для карточек (минимум), `rounded-3xl` для hero-блоков
- `border border-zinc-100` (не zinc-200) для внутренних разделителей
- `shadow-[0_2px_8px_rgba(0,0,0,0.06)]` + `hover:shadow-[0_8px_24px_rgba(0,0,0,0.10)]`
- Primary green — максимум 10% визуального пространства
- Нейтралы только через `zinc-*` (не `gray-*`)
- Иконки только из `lucide-react`

### ⚠️ Текущая проблема дизайна
Сайт содержит **смешанные стили**: часть страниц использует editorial-язык (тёмный hero, mono-шрифт, крупные числа), часть — классический product-дизайн. Нужна унификация по финальному дизайн-файлу.

## 10. Монетизация — план

### 10.1 Направления

#### A. Продвижение объявлений (самое быстрое в реализации)
Продавец платит за повышение видимости своего объявления.

| Пакет | Цена | Что даёт |
|-------|------|---------|
| Топ-10 | 2 990 ₸/7 дней | Объявление в топе каталога + пометка «Топ» |
| Выделение | 1 490 ₸/7 дней | Цветная рамка + пометка «Рекомендуем» |
| Срочно | 990 ₸/3 дня | Пометка «Срочно» + повышение в выдаче |

**Реализация:** поле `promotedUntil: datetime` в Listings + сортировка по нему.

#### B. Подписка для агентств
Ежемесячная плата за инструменты профессиональных участников рынка.

| Тариф | Цена | Что включает |
|-------|------|-------------|
| Базовый | Бесплатно | До 5 объявлений, без значка |
| Профи | 9 990 ₸/мес | До 30 объявлений, значок агентства, страница агентства |
| Бизнес | 24 990 ₸/мес | Безлимит, приоритетная поддержка, аналитика |

**Реализация:** поле `plan` в Users + страница агентства `/agency/[slug]`.

#### C. Безопасная сделка (самый маржинальный)
Юридическое сопровождение за % от суммы сделки.

| Услуга | Цена |
|--------|------|
| Проверка документов | 15 000 ₸ |
| Полное сопровождение | 0.5–1% от суммы сделки |

**Реализация:** форма заявки на `/safe-deal` + CRM для обработки заявок (вначале вручную).

#### D. Платное размещение для физлиц (долгосрочно)
Если рынок вырастет — ввести лимит бесплатных объявлений (напр. 3 активных).

### 10.2 Приоритет запуска монетизации
1. **Сначала:** продвижение объявлений (минимум кода, быстрый доход)
2. **Затем:** форма заявки на безопасную сделку (высокий чек)
3. **Потом:** подписки для агентств (стабильный MRR)

## 13. Известные технические долги

1. **Next.js 16** — Payload пишет "unsupported version" в логах (работает, но риск несовместимости при обновлении)
2. **sharp не установлен** в systemd-окружении — ресайз изображений не работает локально
3. **mock-data.ts** — используется как fallback в странице объявления, нужно убрать
4. **Фильтрация на клиенте** — при росте до 500+ объявлений каталог будет тормозить
5. **Нет тестов** — ни unit, ни e2e
6. **Нет error boundary** — при ошибке рендера страница падает без UX
7. **Нет rate limiting** на API эндпоинтах

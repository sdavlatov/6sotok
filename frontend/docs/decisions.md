> Чтобы не тащить длинную переписку. Новый чат читает этот файл.

## Авторизация (сделано в коде)
- Вход/регистрация через **Google OAuth** + **email/пароль**. **SMS нет** (дорого, альфа-имя).
- Роли при регистрации: **Хозяин**, **Агент/агентство** (бизнес-роль убрана).
- Поля регистрации: роль, имя+фамилия, телефон (обязателен), email, пароль. Повтор пароля и город — убраны.
- Своя валидация (не браузерные пузыри): красная рамка + сообщение под полем.
- Роут: `/api/auth/google`, `/api/auth/google/callback`. ENV: `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`.
- Payload `users.access.create: () => true` (публичная регистрация), поле `role` защищено (только админ).
- Экраны `/login` `/register` — сплит с бренд-панелью (десктоп) + мобильный hero с картой.
- Дизайн auth: классы `.auth-input/.auth-label/.auth-btn-*/.auth-provider` в globals.css, значения 1:1 из макета.

## Позиционирование главной
- **Узкий land-маркетплейс KZ**, НЕ Krisha-клон. Уникальность = земельная экспертиза: границы/форма/рельеф участка, коммуникации (готовность к стройке), юр-статус (госакт/кадастр — со слов продавца, БЕЗ автопроверки, не писать «проверено»).
- Главная = витрина+воронка (инструмент = каталог). Двойной упор **Купить/Разместить**.
- **Масштабируемость:** одинаково хороша на 1 и на 100 000 объявлений. Ленты — empty-safe (пусто → оффер продавцу).
- Монетизация слотами: премиум-лента, бейджи VIP/Срочно, баннер партнёра, витрина застройщиков, тарифы агентств.
- Карту на главной НЕ дублировать (она в каталоге; тут тизер).

## Чипы и бейджи карточки объявления (согласовано, дизайн в работе)

### Платные элементы (3 штуки)

**Реклама** — маркировка платного продвижения позиции в ленте. Обязательная маркировка по закону. Тихий, не конкурирует с контентом. Поверх фото, левый верхний угол.

**Срочно** — платный бейдж-аттрактор. Продавец хочет быстро продать. Поверх фото, правый верхний угол.

**Снижение цены** — платная фича. Продавец вписывает старую цену вручную → карточка показывает старую цену перечёркнутой + новую + разницу в процентах. Не бейдж — изменение блока цены в теле карточки. (История цен автоматически — задача следующего этапа.)

### Бесплатные элементы

**Готов к стройке** — автоматический, система ставит если: `hasElectricity + hasWater + hasRoadAccess + hasStateAct = true`. Главный продуктовый дифференциатор, аналогов у конкурентов нет. В теле карточки под ценой.

**Торг** — бесплатный чекбокс продавца. Выводится как текстовая пометка рядом с ценой, не отдельный чип.

### Правила совместимости
- На фото максимум 2 элемента: Реклама (лево) + Срочно (право)
- Срочно и Снижение цены могут быть одновременно (разные места)
- Делимый участок — решено не показывать на карточке

### Поле в БД
`boost_type: enum('promoted', 'urgent')` — для Рекламы и Срочно.
`price_was: numeric` — для Снижения цены.
Neon SQL при деплое:
```sql
ALTER TABLE listings ADD COLUMN IF NOT EXISTS boost_type varchar;
ALTER TABLE listings ADD COLUMN IF NOT EXISTS price_was numeric;
```

# Хлебные крошки 6соток — инструкция по внедрению

Единый компонент навигационных крошек для всего сайта. Работает и как навигация
для пользователя, и как SEO-разметка `BreadcrumbList` (Google rich results).
Раскладка desktop/mobile — внутри компонента, разного HTML не нужно.

## Что передаётся

- **`site-breadcrumbs.js`** — сам компонент (веб-компонент, Shadow DOM, без зависимостей).
- **`Breadcrumbs (SEO) — единые правила.html`** — визуальная спека (токены, состояния, примеры). Только для справки, в прод не идёт.

## Подключение (один раз на странице)

```html
<script src="site-breadcrumbs.js" defer></script>
```
Ставить после `site-header.js`. Шрифт Inter уже подгружает хедер — отдельно не нужен.

## Использование

Крошки ставятся **сразу под `<site-header>`**, отдельным блоком (не внутри карточек/заголовков).
Путь задаётся атрибутом `trail` — JSON-массив узлов от корня к текущей странице.

```html
<site-header active="buy"></site-header>

<site-breadcrumbs trail='[
  {"name":"Купить","href":"/buy/"},
  {"name":"Алматинская обл.","href":"/almaty/"},
  {"name":"Талгар","href":"/almaty/talgar/"},
  {"name":"Бесагаш · 8 сот."}
]'></site-breadcrumbs>
```

Рендерит: **Главная / Купить / Алматинская обл. / Талгар / Бесагаш · 8 сот.**

### Правила заполнения `trail`
- **«Главная» добавляется автоматически** первым узлом — в `trail` её НЕ включать.
- **Последний узел без `href`** = текущая страница (не ссылка, `aria-current="page"`).
- У всех промежуточных узлов `href` **обязателен**.
- Путь = **реальная иерархия URL**, а не история переходов пользователя.
- `href` можно относительный (`/almaty/`) или абсолютный — в SEO-разметку всё
  превращается в абсолютный `https://6sotok.kz/...` автоматически.

## Где ставить, где нет
- Ставить **со 2-го уровня и глубже**: разделы, регионы, листинги, статьи, карточки.
- **На главной — не ставить.**

## Поведение (уже внутри, ничего делать не нужно)
- **Desktop** (>1024px): одна строка 13px в белой полосе, весь путь виден.
- **Mobile** (≤1024px): 12px, горизонтальный скролл с затуханием у края,
  автопрокрутка к текущей странице. Строка не переносится и не обрезается.
- **Длинный путь (>5 узлов):** середина сворачивается в «…» (раскрытие по клику).
  Скрытые узлы остаются в DOM — SEO-разметка не ломается.

## SEO — что получает поисковик
Компонент сам генерит Schema.org microdata (`BreadcrumbList` / `ListItem` со сквозными
`position`, абсолютные URL, последний узел — тоже `ListItem`, но без ссылки).
Отдельный JSON-LD писать **не нужно** — микроразметки в DOM достаточно, она и есть
источник правды. Проверять через Google Rich Results Test.

## Необязательные атрибуты
- `home="Главная"` — переопределить название корня (по умолчанию берётся язык из
  `localStorage['6sotok:lang']`: ru → «Главная», kz → «Басты бет», en → «Home»).
- `origin="https://6sotok.kz"` — базовый домен для абсолютных URL в разметке.

## Примеры по типам страниц

Листинг региона:
```html
<site-breadcrumbs trail='[{"name":"Купить","href":"/buy/"},{"name":"Алматинская обл."}]'></site-breadcrumbs>
```

Статья журнала:
```html
<site-breadcrumbs trail='[{"name":"Центр знаний","href":"/journal/"},{"name":"Документы","href":"/journal/docs/"},{"name":"5 документов перед покупкой"}]'></site-breadcrumbs>
```

Карточка объявления:
```html
<site-breadcrumbs trail='[{"name":"Купить","href":"/buy/"},{"name":"Алматинская обл.","href":"/almaty/"},{"name":"Талгар","href":"/almaty/talgar/"},{"name":"Участок 8 сот., Бесагаш"}]'></site-breadcrumbs>
```
